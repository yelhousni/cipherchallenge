Signature schemes

Now assume that given an element Q ‚àä ùí¢ it is computationally unfeasible to find k such that Q = k*G. We have seen in a different challenge that it is easy to go from G to k*G but now we are assuming that it is hard to go from k*G to G. This is called a one-way function. In this challenge, we will build a signature scheme based on this function.

Let Mustapha be the signer. He holds a secret signing key k and publishes a public key Q=k*G. To sign a message m, he computes s = k * m (mod p) and sends (m,s).

  1. How can you verify this signature (m,s) knowing just Mustapha’s public key Q?
  2. How would you proceed to recover Mustapha’s secret key k?

Now Mustapha chooses a random number 0 < r < p and computes R=r*G. Then he computes s = r + k * m (mod p) and sends (m,s) alongside R.

  1. How can you verify the signature now?
  2. Can you recover Mustapha’s secret key now?

Now Mustapha signs a different message m' with the same random r. The signature is (m',s') and R.

  1. Can you recover Mustapha’s secret key now? What if Mustapha uses a different random r'?
