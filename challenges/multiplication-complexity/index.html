Time complexity

The time complexity describes the amount of computer time it takes to run an algorithm. It is estimated by counting the number of elementary operations performed by the algorithm. Since an algorithm’s running time may vary among different inputs of the same size, one commonly considers the worst-case time. Algorithmic complexities are classified according to the type of function appearing in the big O notation. Here are some examples:

Name Time complexity
Constant time O(1)
Logarithmic time O(log n)
Linear time O(n)
Linearithmic time O(n log n)
Quadratic time O(n^2)
Polynomial time poly(n)
Exponential time 2^poly(n)

Divide-and-conquer multiplication

Can we do any better?

2-digit numbers

Consider two 2-digit numbers X and Y. Their base-10 representation is X = 10x1 + x0 and Y = 10y1 + y0. Their product is X * Y = (10x1 + x0)(10y1 + y0) = 100x1y1 + 10(x1y0+x0y1) + x0y0.

  1. How many multiplications this product involves?
  2. Can you re-write the middle term x1y0 + x0y1 to re-use x0y0 and x1y1? How many multiplications the product involves now?

n-digit numbers

In general we can write two n-digit number as X = 10^{n/2}x0 + x1 and Y = 10^{n/2}y0 + y1.
